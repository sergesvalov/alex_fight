# Правила и контекст проекта для AI Агентов (Alex Fight)

## Технический стек и ограничения
1. **Движок**: Godot 4.7.
2. **Язык**: GDScript (с использованием статической типизации там, где это возможно).
3. **Платформа**: Основная цель — Android. Все системы управления должны быть адаптированы под Touch (относительный сенсорный ввод для движения, виртуальный джойстик/drag для камеры), с фоллбэками на клавиатуру/мышь для десктопа (уже реализовано в `player_controller.gd`).
4. **Физика**: В проекте используется Godot Jolt Physics. Все настройки `floor_stop_on_slope`, `floor_max_angle` и `floor_snap_length` в CharacterBody3D должны быть выставлены для плавной работы Jolt.
5. **CI/CD**: Проект собирается через Jenkins (`Jenkinsfile`), используя Docker (экспорт в Android APK). Godot запускается в headless режиме. Важно: все изменения в сценах (`.tscn`), скриптах (`.gd`) и ассетах **должны быть закоммичены**, иначе Jenkins соберет старую версию. 
   - **ОСОБОЕ ПРАВИЛО ИМПОРТА АССЕТОВ**: При добавлении новых текстур и файлов извне, **НЕ создавай вручную файлы `.import`** и **НЕ прописывай случайные `uid` в сценах**. Укажи в сцене просто прямой путь (`path="res://assets/..."`) без UID. Движок Godot в headless-режиме на CI-сервере сам увидит новый файл, сам сгенерирует для него `.import` с валидным `.ctex` хешем и сгенерирует UID в процессе сборки. Создание `.import` файлов вручную с фейковыми хешами приведет к сбою импорта! Убедись, что расширение файла соответствует реальному формату (не переименовывай `.jpeg` в `.png`, Godot проверяет magic bytes).
   - **ОСОБОЕ ПРАВИЛО ФОРМАТОВ ИЗОБРАЖЕНИЙ**: Движок Godot при headless-сборке на CI-сервере (Jenkins) может некорректно импортировать сгенерированные `.png` файлы (они становятся белыми текстурами). Все генерируемые ИИ картинки и текстуры **ОБЯЗАТЕЛЬНО должны быть конвертированы в формат `.jpg` (JPEG)** с правильным расширением перед добавлением в проект (используй Python Pillow для конвертации). Не пытайся добавлять в сцены `.png`.

## Архитектура проекта
- `entities/`: Игровые сущности (Player, враги, оружие, интерактивные объекты).
- `scenes/levels/`: Уровни игры (основной уровень: `scenes/levels/hotel_siberia/hotel_level.tscn`).
- `scripts/`: Все скрипты разбиты по категориям (`player`, `ui`, `weapons`, `interactables`).
- `assets/`: 3D модели, текстуры и звуки.
- `hud/`: Интерфейс пользователя и виртуальные элементы управления.

## Правила работы (Behavioral Rules)
- При создании новых ассетов (сцен, текстур), всегда убеждайся, что в сценах прописаны валидные `uid` для внешних ресурсов.
- Используй `replace_file_content` и `multi_replace_file_content` для точечных изменений вместо полной перезаписи.
- Завершив задачу, всегда делай коммит и пуш (`git add . && git commit -m "..." && git push origin main`), чтобы Jenkins мог сразу собрать билд для проверки.
