# Alex Fight

> 3D-шутер | Godot 4.7 | Android / PC | Ретрофутуризм | P.T.-style Horror

Коридорный шутер с элементами survival horror и психологического триллера. Действие разворачивается в заброшенной провинциальной гостинице «Сибирь», которая превратилась в сюрреалистическую бесконечную петлю. Герой — полицейский без памяти (Алекс) — ищет выход на крышу, собирает голографические VHS-кассеты с отголосками своего прошлого и отбивается от жуткого проекта «Цербер».

---

## 🌟 Ключевые особенности

- **Бесконечная петля и Этажи (Non-Euclidean Geometry)**: 3 и 4 этажи гостиницы. Лестничная клетка работает как физический портал — бесшовная телепортация создаёт пугающий эффект лабиринта в духе P.T.
- **Таинственные Двери-Порталы**: после сбора всех VHS-кассет этажа в стене комнаты прорезается таинственный выход — телепорт на другой этаж с перемешиванием кассет и монстра.
- **Лазерное оружие**: научно-фантастический лазерный пистолет с механикой нагрева/остывания.
- **Визуальные эффекты**: перманентные обугленные следы на стенах (технология Decal/Multiply).
- **Компонентная архитектура**: HUD, AI и управление разбиты на независимые модули, общающиеся через глобальный `EventBus`.

## 🚀 Быстрый старт

1. Открыть проект в **Godot 4.7**
2. Импортировать (`Project → Import`)
3. Запустить `scenes/levels/hotel_siberia/base_hotel_level.tscn`
4. Для Android: `Project → Export → Android`

> **⚠️ ВАЖНО ДЛЯ CI/CD**: Godot Headless на Jenkins сам генерирует `.import`.
> - **Никогда** не создавайте `.import` файлы вручную с фейковыми UID.
> - Все текстуры — строго **`.jpg` (JPEG)**: PNG может стать белым на headless-сборке.

## 🗺️ Система координат уровня Hotel Siberia

### Мировые оси

| Ось | `+` | `−` | Роль на уровне |
|-----|-----|-----|----------------|
| **X** | Восток | Запад | Поперёк коридора / глубина комнат |
| **Y** | Вверх | Вниз | Высота |
| **Z** | Юг | **Север (вглубь)** | Длина коридора |

> Уровень вытянут по **−Z**. `(0, 0, 0)` — вход в коридор у лобби. Коридор тянется от Z=0 до Z=−85.

### Высотная шкала

| Точка | Y | Описание |
|-------|---|----------|
| Центр пола (CSGBox) | `−0.25` | Пол толщиной 0.5 м |
| **Игровой пол** | `0.0` | Поверхность передвижения |
| Центр стен | `2.0` | CSGBox стен высотой 4 м |
| Нижняя грань потолка | `4.0` | |
| Центр потолка (CSGBox) | `4.25` | |
| Петля двери | `0.0` | Всегда на игровом полу |
| Дверной проём (высота) | `0.0 .. 2.5` | Полотно двери 2.5 м |

### Поперечное деление по X

```
 X=-13.7         X=-3.5     X=0     X=+3.5         X=+11.3
    ┌───────────────┐         │         ┌───────────────┐
    │ Двойные ном.  │  КОРИ  ДОР  │  Одиночные ном. │
    │   (10.8×12 м) │  X: -3.5..+3.5  │   (8.4×7.2 м) │

    └───────────────┘         │         └───────────────┘
      CorrWallW X=-3.5        │        CorrWallE X=+3.5
```

### Формулы: локальные → мировые координаты

```gdscript
# Объект в двойном номере Ln (local_x, local_z из double_room.tscn):
world_x = local_x + (-8.3)
world_z = local_z + Z_L(n)      # Z_L(n) = 4.0 - (n-1)*12.0

# Объект в одиночном номере Rn (local_x, local_z из single_room.tscn):
world_x = local_x + 7.1
world_z = local_z + Z_R(n)      # Z_R(n) = -3.6 - (n-1)*6.0
```

### Дверные проёмы в стенах коридора

Центр проёма по Z = **Z_центр_комнаты + 0.5** (для обоих типов комнат).
Зазор в стене коридора: **±1.0 м** от центра проёма.

| Дверь | Стена | Плоскость двери | Размер проёма (Ш×В) |
|-------|-------|-----------------|----------------------|
| DoubleRoom MainDoor | WallE, X=4.0 | YZ | 1.2 × 2.5 м |
| DoubleRoom WCDoor | WC_WallW, X=1.75 | YZ | 0.8 × 2.5 м |
| SingleRoom MainDoor | WallW, X=−3.0 | YZ | 1.2 × 2.5 м |
| SingleRoom WCDoor | WC_WallS, Z=−0.25 | XY | 0.8 × 2.5 м |

### Объект `entities/props/door.tscn`

```
DoorRoot  ← позиция инстанса (петля на полу Y=0.0, у края проёма)
└── Hinge  ← ось вращения
    └── AnimatableBody3D  offset=(+0.5, +1.25, 0) от петли
        └── BoxMesh  1.0 × 2.5 × 0.1 м  (ширина × высота × толщина)
```

**Правило ориентации:**
- Стена ⊥ **X** (коридорные двери) → поворот **±90° вокруг Y**
- Стена ⊥ **Z** (санузел SingleRoom) → **identity** (без поворота)
- Петля всегда у **ближнего края** проёма, `Y = 0.0`

## 📝 Changelog

### 2026-07-01 — Багфиксы коллизий и генератора

#### `hotel_level_generator.gd`
> **Проблема:** Разрыв сетки между одиночными номерами и коридором. 
> **Исправление:** Инстансы `SingleRoom` сдвинуты на `X=7.1`, чтобы их стены идеально стыковались с краем коридора `X=3.5`.
> **Улучшение:** Добавлена бесшовная телепортация на лестнице (копия этажа снизу/сверху), а стены-заглушки убраны.

#### `player_interaction.gd` & `player.tscn`
> **Проблема:** Невозможно подобрать VHS-кассеты на полу.
> **Исправление:** `RayCast3D` перенесён из корневого узла игрока в `CameraRig/Camera3D`, чтобы луч взаимодействия корректно наклонялся по оси X (pitch).

#### `double_room.tscn` / `single_room.tscn`
> **Проблема:** Игрок застревал в дверном проёме открытой двери.
> **Исправление 1:** Устранена проблема `internal edge collision` движка Jolt. Пол внутри комнат `Floor` укорочен по оси X так, чтобы он не пересекался с полом коридора внахлёст.
> **Исправление 2:** Операция CSG-вычитания (`MainDoorHole`) убрана. Стены с дверями (`WallE` / `WallW`) разбиты на 3 прямых физических блока (`_L`, `_R`, `_Top`), гарантируя наличие физического прохода.
