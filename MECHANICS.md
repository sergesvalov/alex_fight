# Механики Игры «Alex Fight»

В этом документе описаны основные игровые механики, которые формируют ядро геймплея 3D-шутера с элементами survival horror.

## 1. Базовое перемещение и управление
- **Мобильный интерфейс (Android):** 
  - Левая половина экрана: **Невидимая зона относительного сенсорного ввода**. При касании и свайпе вычисляется вектор смещения, который задает направление и скорость движения игрока. Вектор ограничен максимальным радиусом и сбрасывается при отпускании экрана.
  - Правая половина экрана: **Управление камерой** (взгляд/прицеливание) через глобальный менеджер `MouseManager`.
  - **Двойной тап**: Быстрое двойное касание по экрану вызывает выстрел из лазерного пистолета.
- **Десктоп-фоллбэк:**
  - При запуске на Windows/macOS/Linux мышь автоматически захватывается (Captured). 
  - Для движения используется классическая раскладка клавиатуры (WASD / Стрелки), левая кнопка мыши — для выстрела.
- **Физика перемещения:** Персонаж (CharacterBody3D) учитывает гравитацию и угол наклона поверхностей (до 45 градусов), плавно останавливаясь на склонах (используется Godot Jolt Physics).

## 2. Боевая система и Raycast Gunplay
- **Стрельба (Raycast):** Оружие мгновенно выпускает луч от центра экрана (Camera3D) вглубь сцены.
- **Определение попаданий и Декали:** Если луч пересекается со стеной или врагом, в точке попадания инстанцируется префаб `hit_marker.tscn`.
  - Для соответствия строгим требованиям CI-пайплайна Jenkins (неподдерживаемые PNG), декали реализованы через `QuadMesh` + `StandardMaterial3D` с режимом смешивания **Multiply (Blend Mode = 3)**. Это позволяет использовать компактные черно-белые JPG-текстуры без альфа-канала для создания реалистичных перманентных ожогов от лазера на стенах.
  - Попадание также вызывает сноп искр (`GPUParticles3D`).
- **Game Feel (Микро-отдача):** При каждом выстреле камера резко приподнимается вверх на несколько градусов с помощью Tween и плавно возвращается обратно за доли секунды.
- **Механика перегрева оружия (Лазерный пистолет):**
  - Оружие не использует патроны, а накапливает тепло (от 0 до 100).
  - Каждый выстрел повышает температуру. В мобильном интерфейсе (HUD) отображается шкала нагрева (HeatBar). Состояние нагрева отправляется в HUD через глобальную шину `EventBus`.
  - Если игрок не стреляет, оружие плавно остывает.
  - Если температура достигает 100, активируется блокировка (Overheat): стрелять невозможно, пока оружие полностью не остынет до 0.

## 3. Левел-дизайн и Психологический Хоррор (P.T. Loop)
- В игре применяется **неевклидова геометрия (Non-Euclidean Geometry)** для создания параноидальной атмосферы.
- На главной локации (отель «Сибирь») расположена лестничная клетка, работающая в режиме **бесконечной петли**.
- При подъеме или спуске по лестнице, невидимые триггерные зоны (`Area3D`) мгновенно и бесшовно смещают (телепортируют) позицию игрока на высоту одного этажа. Поскольку все 3 этажа лестничной клетки абсолютно идентичны, игрок не замечает перехода и оказывается запертым в бесконечном лабиринте коридора, который всегда возвращает его к началу.

## 4. Взаимодействие с миром (Интерактивность)
- В центре экрана расположен `RayCast3D` короткого действия (длина луча ~3 метра).
- Для мобильных устройств в HUD добавлена кнопка **[INTERACT]**.
- Поднятые предметы (голографические кассеты) обновляют счетчики в модульном UI (TapesCounter) через сигналы `EventBus`.
- **Зеркала**: В отеле есть зеркала, которые используют `ReflectionProbe` для отражения тела игрока. Зеркало также работает как нарративный триггер — при приближении к нему система диалогов выводит текстовые мысли Алекса на экран.

## 5. Враги: ИИ «Цербер»
- Проект «Цербер» использует модульный искусственный интеллект, разделенный на подкомпоненты: `CerberusMovement` и `CerberusSensors`.
- ИИ базируется на Finite State Machine (IDLE, PATROL, CHASE, ATTACK, RETURN).
- Использует `NavigationAgent3D` для навигации, зону обнаружения (`Area3D`) и Line of Sight (`RayCast3D`).
- При переходе врага в режим CHASE, музыка переключается на боевую (`combat`), а состояние игры меняется через `GameStateManager`.

## 6. Освещение и атмосфера
- Игра использует темное, мрачное освещение. Солнечный свет сведен к минимуму.
- Игрок оснащен **наплечным фонариком** (`SpotLight3D`), привязанным к камере, который позволяет ориентироваться в темноте и создает эффект клаустрофобии.
